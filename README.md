## My Philosophy

I don't believe in gatekeeping software development.

You don't need to memorize every framework.

You don't need to know every algorithm.

You don't need to pretend you wrote every line yourself.

You need to understand the problem.

You need clear thinking.

You need good taste.

You need persistence.

The rest can be learned—or collaborated on.
